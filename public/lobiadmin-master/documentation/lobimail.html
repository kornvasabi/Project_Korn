LobiMail

LobiMail is email interface jquery plugin exclusively written for LobiAdmin

Usage

LobiMail contains 5 views.

  • Main
  • Inbox
  • Sent
  • Message view
  • Compose view

Put Main View html into your page or load it by ajax and call plugin

<div id="mailbox" class="fadeInDown animated">
    ...
</div>
<script>


    $('#mailbox').lobiMail();        

</script>

See full example bellow

Main view

Configure search

Plugin searches <form> inside.col-search so make sure you have element with .col-search class and <form> in it.

Whenever you submit the search form if compose view or email view is shown nothing will happen but if any other view is shown such as inbox, sent, draft, ect. The same view will be loaded but search value will be sent as search parameter. You can easily configure back and real time search will be applied. 

<div class="col-search">
    <form>
        <div class="input-group">
            <input type="text" name="search" class="form-control" placeholder="Search">
            <span class="input-group-btn">
                <button class="btn btn-default">OK</button>
            </span>
        </div><!-- /input-group -->
    </form>
</div>

Configure compose button

Plugin relies on .btn-compose class to find compose button. DO NOT remove this class.

<a href="Put compose view url here" class="btn btn-info btn-block btn-lg btn-compose btn-pretty">Compose</a>

Configure Menu

Plugin relies on .nav-menu class to find menu. DO NOT remove this class.

Plugin relies on data-type="" attribute of <a> element to load view properly, so please make sure you include it for every menu item.

Inbox view

If you have unread emails also add data-unread-emails="Number of unread emails" attribute.

"Select all" checkbox

In order select all checkbox to work properly make sure you added data-action="select-all" to checkbox.

The selector data-action="select-all" can be changed from plugin options.

<button class="btn">
    <label class="checkbox-inline lobicheck">
        <input type="checkbox" data-action="check-all">
        <i></i>
    </label>
</button>

Select actions

In order select actions (select read, select unread, etc.) to work properly use .dropdown-select-type class on <ul> and add data-select="" attribute (available options: "all", "none", "read", "unread", "starred", "unstarred") to <a> element inside <li> 

<ul class="dropdown-menu dropdown-select-type" role="menu">
    <li><a href="#" data-select="all">All</a></li>
    <li><a href="#" data-select="none">None</a></li>
    <li><a href="#" data-select="read">Read</a></li>
    <li><a href="#" data-select="unread">Unread</a></li>
    <li><a href="#" data-select="starred">Starred</a></li>
    <li><a href="#" data-select="unstarred">Unstarred</a></li>
</ul>

Emails

For opening email you need to add data-key="" to every <tr> and data-view-email-url="" attribute to <table>.

data-view-email-url="" attribute may be added to outer most div .lobimail also (In this case <table>'s data-view-email-url="" attribute will override .lobimail's data-view-email-url="") or can be set from javascript.

On the server side use key variable of GET request for loading single email.

Use .unread, .starred classes on <tr> to mark email as unread or starred.

Also you will have to change fa fa-star-o to fa fa-star if you mark the email as starred

<table class="table" data-view-email-url="Single email url">
    <tbody>
        <tr class="unread" data-key="1">
            <td class="td-checkbox">

Deleting emails

For deleting emails [data-action="delete"] element is used. If this element contains href attribute different from "#"delete request will be send on this url. Otherwise deleteUrl option will be used from LobiMail options. If neither this url is provided pressing delete button simple will do nothing. When sending delete request table row data-key attribute is used for email identification.

Example

Compose
  • Inbox
  • Sent
  • Draft
  • Spam
  • Archive

Options

//Boolean. 
enableSearch: true,
//Boolean. Activate check all email button
enableSelectAll: true,
//Boolean. Show debug messages
enableDebug: false,
//Boolean. Enable to mark email es starred
enableStar: true,
//Boolean. Enable tooltips
enableTooltips: true,
//String. View name which will be loaded when plugin is initialized
defaultView : 'inbox', //compose, email, inbox, sent, draft, archive, trash ...
//Integer. This option is used if defaultView is 'email'. This integer value will be used for loading email
defaultMessageKey: false, //Number. Key of the message you want to load by default
//String. The url which will be used for deleting emails
deleteUrl: '',
//String. The url which will be used for loading single email. 
//Outer most div (.lobimail) element data-view-email-url="" attribute will override this option 
//if it is provided. <table> data-view-email-url="" attribute will override both.
//So you have three way to specify viewEmailUrl attribute.
viewEmailUrl: '',
//String. The url which will be used for marking email as starred or unstarred. 
//On server side you get "ids[]" array (containing keys of email) and "star" (true/false) to mark emails starred or unstarred
starEmailUrl: '',

//String. Selector for go back button when email is opened
goBackBtn: '[data-action="go-back"]',
//String. Selector for select all button
checkAllBtn: '[data-action="select-all"]'
